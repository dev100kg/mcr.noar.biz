# 中途採用ランディングページ運用 セキュリティ整理

---

## 【結論】

**一般的な企業採用サイトと同等以上のセキュリティ水準と判断しています。**
現時点で重大な技術リスクは確認されておらず、適切な運用を前提に利用可能と考えています。

なお、セキュリティはゼロリスクにはできないため、
**現実的なリスク管理を前提に継続的に確認・改善していく方針です。**
追加観点があれば柔軟に取り入れますので、ご指摘ください。

---

## 【インフラ観点】

### サイト構成

静的サイトホスティングを採用し、攻撃面積を最小化しています。

- 静的サイトホスティング（Netlify）
  - HTML / CSS / JSのみで構成
  - サーバーサイド処理なし
  - データベースなし
  - Netlify Forms利用を今後検討
    - クラウド基盤側での保存データ暗号化（AES-256相当）

- 本社ホームページとは分離構成
  - ドメイン管理：Amazon Route53（勤怠経費システムと共通）
  - ソースコード管理：GitHub（完全に独立）
  - 作業管理：GitHub Issues / Pull Requests（完全に独立）
  - ホスティング：Netlify（完全に独立）

自社サイトとは物理的・論理的に分離しており、
万一の障害・侵害があっても影響範囲は限定的です。

なお、過去にサブドメイン設定誤りによる表示障害が発生しましたが、
想定される影響は表示停止程度に留まる構成です。

---

### クラウド基盤

サーバー直接公開型と比較して攻撃面積が小さい構成です。

- セキュリティ監査・認証
  - ISO27001認証取得環境
  - SOC2監査対象サービス
- CDN配信型ホスティング
  - 元サーバー非公開による侵入リスク低減
  - 分散配信によるDDoS耐性向上
  - SSL証明書管理・脆弱性対策を基盤側で継続実施

---

## 【運用観点】

### 現在の運用状況

- 現時点では個人情報収集は未実施
- 管理アカウントは多要素認証（MFA）導入済
  - AWS：済
  - GitHub：済
  - Netlify：済
- 管理アカウント共有なし
- GitHub Pull Requestベースのリリース運用
  - 承認なしで公開されない仕組み

---

### 今後のフォーム設置時の対応予定

通信仕様：

- HTTPS（TLS1.2以上）による通信暗号化
- Netlify基盤側での保存データ暗号化（AES-256相当）
- 常時SSL配信

想定用途：

- カジュアル面談申込フォーム
- 求人応募フォーム

導入時の対応：

- プライバシーポリシー整備
- Netlify Formsによるデータ収集
- XSS / CSRF等の基本的入力対策（プラットフォーム標準機能）
- 応募目的限定利用の明示
- 添付ファイル受付なし
- 動的サーバー処理なし
- 通知メールの適切管理（リンク確認・不要データ削除）

---

## 【責任分界の整理】

### クラウド側責任

- インフラセキュリティ
- 通信暗号化
- データ保存暗号化
- CDN防御
- インフラ監視・パッチ管理

### 自社運用責任

- アカウント管理（MFA等）
- 応募データ取り扱い
- プライバシーポリシー整備
- アクセス権限管理
- 運用ルール遵守

---

## 【総合評価】

**採用LP用途として適切な安全水準と判断しています。**

- インフラは大手クラウド水準
- 静的サイト構成により攻撃面積が小さい
- 本体システムとは分離構成
- 最大リスクは技術より運用管理領域
